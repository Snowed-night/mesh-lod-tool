# 答辩 PPT 提纲

## 1. 标题页

误差驱动的三维网格 LOD 简化与可视化系统

建议副标题：

```text
自实现 QEM 基线 + meshoptimizer 工程后端 + 屏幕空间误差自适应切换
```

## 2. 背景与问题

- 复杂三维模型面数高，加载和渲染成本大。
- 固定比例 LOD 缺少视觉误差依据。
- 课程项目需要兼顾算法理解、工程实现和展示效果。

## 3. 系统整体架构

展示数据流：

```text
OBJ/3MF 模型 -> C++ LOD 生成 -> CSV 统计 -> Web 可视化 -> 自适应切换/热力图
```

## 4. 核心算法 1：QEM 基线

- 点到平面的距离平方和。
- 边折叠代价。
- 边界保护和法线翻转检测。

## 5. 核心算法 2：工程后端

- 引入 meshoptimizer。
- 复杂模型上更快、更稳定。
- 保留自实现 QEM 作为原理基线。

## 6. 创新点 1：误差驱动自适应 LOD

- 使用几何误差和屏幕空间误差。
- 根据像素阈值自动选择最低复杂度 LOD。
- 对比固定距离切换。

## 7. 创新点 2：视觉感知热力图

- 蓝色低风险、黄色中风险、红色高风险。
- 结合局部结构敏感度、轮廓显著性和 LOD 简化程度。
- 不把屏幕误差直接叠进颜色，避免放大后整片变红。

## 8. Web 展示效果

- 上传 OBJ/3MF。
- 手动 LOD / 自适应 LOD。
- PBR 材质、光源控制、接触阴影。
- 误差-复杂度曲线和统计表。
- 平滑法线和阴影修正，减少复杂模型表面条纹。

建议放：

```text
result/report_assets/composite/fig03_girl_lod_grid.png
```

这一页重点展示系统不是只生成 OBJ 文件，而是能在 Web 端进行线框、无材质、真实材质和热力图对比。

## 10. 总结与未来工作

- 完成从算法基线到工程工具的闭环。
- 支持真实复杂模型。
- 后续可加入更严格距离评估、多格式支持和自动视频导出。

## 答辩时重点说法

1. “我没有把项目定位成单一算法复现，而是做成了一个完整 LOD 工具链。”
2. “自实现 QEM 用来说明原理，meshoptimizer 用来支撑真实复杂模型。”
3. “创新点主要在误差驱动的自适应切换和可解释热力图，而不是声称自己超过成熟简化库。”
4. “屏幕空间误差用像素衡量当前视角下的可见误差，因此比固定距离切换更合理。”
5. “热力图是实时结构风险可视化，不等价于严格 Hausdorff 距离，这是我主动说明的边界。”
