#### QEM原理理解：
QEM 就是寻找一个点，使它到两个端点原来相邻三角面所在平面的距离平方和最小。然后把两个点合并成那个点，从而减少一个顶点，减少一个或两个三角面。

需要最小化新点到所有相关平面的距离平方和，来求 v_bar 这的最优的点。
写成公式就是：
sum distance(v, p)^2
这里：
v 是候选新点；
p 是某个原始三角面所在的平面；
不是求最小值本身，而是求“让这个函数最小的那个变量 v_bar”。

点v (x, y, z) 到平面p（ax + by + cz + d = 0）的有符号距离是：
ax + by + cz + d
为了写成矩阵形式，引入：
v = [x, y, z, 1]^T
p = [a, b, c, d]^T
distance(v, p)^2 = (v^T p)^2= v^T p p^T v
记：Kp = p p^T
所以：distance(v, p)^2 = v^T Kp v

一个顶点 vi 可能属于多个三角面，记顶点到多个三角面的距离和为：
Qi = sum Kp
对于一条边 (v1, v2)有两个顶点
所以：Q = Q1 + Q2
最终优化目标就是：
v_bar = arg min_v v^T (Q1 + Q2) v
求 v_bar 使得 v^T (Q1 + Q2) v 最小化，也就是求一个点使其到两个端点原来相邻三角面所在平面的距离平方和最小。

算法完整思路：
1、每个面求平面方程 p=[a,b,c,d]
2、将每个面构造误差矩阵方便计算（误差指的是距离平方和） Kp = p p^T
3、每个顶点累加相邻面的 Kp，得到 Qi
4、对每条边 (i,j)，计算 Q = Qi + Qj
5、遍历求最优新点 v*，使得E(v) = v^T Q v 最小化，也就是求一个点使其到两个端点原来相邻三角面所在平面的距离平方和最小
（优先通过解线性方程求最优点；如果矩阵不可逆，就在 vi、vj、边中点这几个候选里选误差最小的。）
6、把代价最小的边先折叠并重复

## 阶段 8：支持 3MF 自动转 OBJ

3MF 是 3D Manufacturing Format，意思是“三维制造格式”。它常见于 3D 打印模型，本质上是一个压缩包，里面包含 XML 格式的模型描述和可能的外部对象文件。

OBJ 是当前 C++ 网格简化工具直接支持的格式。为了让下载的 3MF 模型也能进入同一套 LOD 管线，Web 后端增加了 3MF 转 OBJ。

实现位置：

- [server.js](E:/学习比赛/课程编程/计算机图形学/web/server.js)

核心步骤：

```text
读取 .3mf 文件
    ↓
用 JSZip 解压
    ↓
找到 .model XML 文件
    ↓
解析 vertices 和 triangles
    ↓
处理 component 引用和 transform 变换
    ↓
写出 source.obj
    ↓
调用 C++ LOD 生成工具
```

这里的 transform 是“变换矩阵”，用来表示模型组件的平移、旋转、缩放关系。3MF 里一个模型可能不是一个直接网格，而是多个组件拼起来的，所以转换时要把组件顶点变换到最终位置。

为了避免大模型转换时内存占用过高，`writeObjFromMeshes` 没有先拼一个巨大字符串，而是用 `fs.writeSync` 逐段写入 OBJ 文件。

转换结果保存位置：

```text
result/web/models/<job>/source.obj
```

如果上传的 3MF 能在 `素材/下载模型/` 中找到同名同内容文件，转换出的 OBJ 会同步复制回原模型目录，方便下次直接使用。

## 阶段 10：重复上传去重和缓存复用

早期 Web 结果目录按时间戳生成，重复上传同一个模型会产生很多内容几乎一样的目录，后面清理起来很混乱。

后来改为用文件哈希去重。哈希可以理解成“文件内容指纹”：只要文件内容完全一样，算出来的字符串就一样。

当前做法：

- `server.js` 用 SHA-256 计算上传文件内容哈希。
- 任务目录名采用 `<模型名>_<哈希前12位>`。
- `result/web/jobs.json` 记录哈希到任务目录的映射。
- 再次上传同一个文件时，直接复用已有 LOD 和统计表。
- 上传临时文件使用内存存储，处理结束后清理临时目录。

这样解决的问题：

- 不再反复生成一堆时间戳目录。
- 大模型重复测试时不需要重新简化。
- Web 页面显示的输出路径和磁盘实际目录更容易对应。

## 阶段 12：误差驱动的自适应 LOD 切换

老师指出项目完整但创新点不足后，项目从“能生成和查看 LOD”进一步改成“能根据视觉误差自动选择 LOD”。

这里的屏幕空间误差，意思是：模型简化造成的几何误差投影到屏幕上以后大约占多少像素。它比单纯按三角面比例切换更合理，因为同一个几何误差在近处更明显，在远处更不明显。

参考依据：

- QEM 论文关注“误差度量”和“面数减少”的关系。
- meshoptimizer 提供简化后的 `result_error`，表示相对模型尺度的几何偏差。
- 3D Tiles/Cesium 这类实时三维系统使用 Screen-Space Error，也就是屏幕空间误差，来决定是否切换更高精度层级。

这次新增的功能：

- 保留原来的手动 LOD 模式。
- 新增自适应 LOD 模式，通过按钮切换。
- 增加屏幕误差阈值滑块，单位是像素。
- C++ 统计表新增 `normalized_error` 和 `absolute_error`。
- Web 前端根据相机距离、视场角、画布高度和几何误差估计当前 LOD 的屏幕误差。
- 自动选择满足误差阈值的最低复杂度 LOD。
- 底部表格增加压缩率、几何误差、屏幕误差、耗时。
- 增加误差-复杂度曲线，用于展示不同 LOD 在模型复杂度和几何误差之间的关系。

核心公式来自 meshoptimizer 文档中的 LOD 选择思路：

```text
1 像素对应的模型空间长度 =
distance * (2 * tan(fovy / 2) / screen_height)

屏幕空间误差 =
几何误差 / 1 像素对应的模型空间长度
```

当前代码对应：

## 阶段 13：视觉感知热力图和电影化渲染展示

老师继续反馈“创新性还不够、视觉效果不够惊艳”后，项目又补了一层展示和分析能力。

这次不是继续堆格式支持，而是围绕两个目标改：

- 让项目能讲出更明确的图形学创新点。
- 让最终视频有更好的光影、材质和可视化冲击力。

新增功能：

- 增加“无材质”、“真实材质”和“误差热力图”三个显示模式。
- 真实材质模式使用 PBR 材质、环境光贴图、ACES 色调映射和接触阴影。
- 误差热力图用颜色显示当前 LOD 在视觉上更容易出问题的区域。
- 热力图颜色规则：蓝色表示风险低，黄色表示中等，红色表示风险高。
- 增加热力图强度滑块，方便不同模型下调整颜色对比度。
- 地面阴影根据模型包围盒自动适配，不需要针对每个模型手调。
- 模型网格开启投射阴影和接收阴影，减少“模型漂在空中”的感觉。

这里的 PBR 是 Physically Based Rendering，意思是“基于物理的渲染”。它用粗糙度、金属度、环境反射等参数让材质更接近真实物体。

这里的 ACES 色调映射，是一种影视和实时渲染里常用的颜色映射方法。简单理解就是：让亮部和暗部更像摄影机拍出来的效果，不会只是普通线性明暗。

这里的误差热力图不是精确的逐点 Hausdorff 距离。Hausdorff 距离是衡量两个几何集合之间最大偏差的一种严格指标，但实时计算复杂度比较高。当前采用的是更适合交互展示的近似指标：

```text
热力图风险 =
局部结构敏感度
+ 当前视角轮廓显著性
+ LOD 简化程度权重
```

其中：

- 局部结构敏感度主要来自局部法线变化和边界信息，尖锐边、薄片、复杂褶皱更容易变红。
- 当前视角轮廓显著性表示接近模型外轮廓的区域，人眼更容易注意。
- LOD 简化程度权重表示低精度模型下风险颜色会更明显，但不会因为相机放大就整片变红。

屏幕空间误差仍然用于自适应 LOD 选择和表格指标，但不再直接叠加到热力图颜色里。原因是：相机越近，屏幕空间误差一定会变大，如果直接叠进颜色，会出现“放大后整片变黄/变红”的误导现象。

后续根据真实模型测试又调整了一次热力图参数：降低边界权重，拉长蓝色区间，并把默认强度设为 0.65。这样像植物叶片这种薄片模型不会因为到处是边界而整片红黄，放大视角也不会直接改变颜色等级。

这次改动集中在：

- [index.html](E:/学习比赛/课程编程/计算机图形学/web/public/index.html)：新增显示模式按钮和热力图指标说明。
- [main.js](E:/学习比赛/课程编程/计算机图形学/web/public/main.js)：新增 PBR 场景、环境光、阴影、热力图顶点颜色计算。
- [styles.css](E:/学习比赛/课程编程/计算机图形学/web/public/styles.css)：新增显示模式按钮样式。

这个阶段可以作为新的创新点来讲：

```text
项目不只生成多级 LOD，还将局部结构敏感度和视觉显著区域结合起来，用热力图解释哪些区域在低精度模型中更容易被人眼察觉。
同时使用 PBR 和环境光照增强展示效果，让算法结果更适合视频展示和答辩演示。
```

后续根据光源测试又做了一次渲染模式修正：

- 增加“无材质”模式，使用普通漫反射材质，主要用于观察点光源颜色、方向和强度。
- 保留“真实材质”模式，但降低环境反射、整体曝光和默认自然光强度，避免模型本身过亮导致点光源效果不明显。
- 保留“误差热力图”模式，用于解释结构敏感区域。

这里的“真实材质”目前更准确说是统一 PBR 展示材质，并不是完整读取 OBJ 配套 MTL 贴图。它的作用是给报告和视频提供更接近真实光照的表面效果；如果后续要进一步增强真实感，可以加入 MTLLoader 或 glTF 模型支持。

后续又根据界面使用体验做了一次控制台重排：

- 左侧只保留标题、上传、状态、路径和操作提示。
- 中间保留完整高度的模型视窗。
- 右侧放 LOD 控制、显示模式、光源、误差曲线和 LOD 表格。
- 将“误差热力图”按钮缩短为“热力图”，避免按钮文字换行。

后续在 Girl 模型上发现无材质和真实材质模式会出现明显波浪状条纹。主要原因有两个：

- 模型网格既投射阴影又接收自己的阴影，复杂曲面上容易出现 shadow acne。shadow acne 是阴影贴图精度不足时，表面错误遮挡自己形成的条纹。
- OBJ 加载后有些位置相同的顶点没有合并，直接计算法线会让相邻三角面之间不够连续。

修正方式：

- 模型本体只 `castShadow`，不再 `receiveShadow`，地面仍然接收阴影。
- 使用 Three.js 的 `mergeVertices` 合并重复顶点，再重新计算平滑法线。
- 给方向光和点光源增加随模型尺度变化的 `normalBias`，降低阴影自相交。

现在展示时建议：

```text
无材质：调光和观察光源效果。
真实材质：录制最终外观展示。
误差热力图：讲创新点和误差可解释性。
```
